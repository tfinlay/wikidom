WikiDom by @tfinlay

An "I'm feeling lucky"-style search engine that takes the name of a website and redirects you to the website itself using the power of Wikipedia.

Inspired by @aaronjanse's WebExtension implementation (dns-over-wikipedia) and discussion on Hacker News. The primary differentiator is that this implementation requires no extension to be installed.

Explanation

Wikipedia articles often contain official URLs of websites. This website allows you to provide the name of an article, and will automatically look up the URL associated with that name in Wikipedia. If one is found, you will be redirected to that URL!
All of this is done in your browser, so information about your search queries is not visible to me.

Obviously, since it's Wikipedia, the URL you're directed to cannot be guaranteed to be correct all the time, so use at your own risk!

Usage

To begin using this site, simply add #q=<your query here> on to the end of the current URL.

For example, to search for "Hacker News", go to , which will correctly redirect you to https://news.ycombinator.com/

Convenience

If you want to start using this site more regularly, you can add this site as a search engine to integrate it more into your browser.
In some browsers, you can then choose to search with this alternate search engine some of the time.

Firefox:

  1. Add this page as a search engine by following these instructions.
  2. Optionally, go to search settings, choose WikiDom, and double click on the "Keyword" cell next to it to assign it to a keyword (e.g. wikidom)
  3. Then, to search with WikiDom instead of your default search engine, type @wikidom before your query. If you chose not to assign it a keyword, you can also select it through the GUI.

Other browsers: I don't know a nice built-in way to do this (please let me know if you do!), so I suggest adding the link below to your bookmarks bar. You can click it on non-protected pages to search!
